1. Visão Geral

StayAlive responde à pergunta central:

"Se eu gastar X hoje, vou ficar a negativo nos próximos 30 dias?"

A app permite ao utilizador:

  • Introduzir o saldo bancário atual manualmente
  • Importar extratos bancários em CSV (CGD, Millennium, Santander, Novo Banco, BPI)
  • Categorizar transações automaticamente via regras keyword
  • Registar despesas fixas e rendimentos recorrentes
  • Ver uma projeção diária de saldo a 30 dias
  • Simular um gasto em tempo real sem escrever na base de dados
Público-alvo: utilizador individual (single-user), contexto mobile on-the-go.
MVP scope: 1 conta bancária por utilizador; sem sincronização automática com bancos.

3. Data Model & Auth

Auth Flow

User clica "Entrar com Google"
  → Supabase Auth redireciona para Google
  → Callback → sessão JWT gerada
  → profiles row criado automaticamente (DB trigger)
  → User entra no dashboard
  • JWT guardado em cookie httpOnly; Next.js middleware valida em cada request
  • RLS em todas as tabelas: USING (user_id = auth.uid())

4. Onboarding Wizard

Wizard linear de 5 passos com barra de progresso no topo. Estado guardado em localStorage até ao passo final (commit para Supabase apenas no "Concluir"). Pode ser abandonado e retomado.

[ 1 Saldo ] → [ 2 CSV ] → [ 3 Mapeamento ] → [ 4 Categorias ] → [ 5 Fixas ] → Dashboard

Botão "Saltar" disponível em todos os passos exceto o Passo 1 (saldo é obrigatório).

Passo 1 — Saldo Atual

  • Input numérico; aceita vírgula ou ponto decimal; pode ser negativo
  • Guardado em profiles.balance

Passo 2 — Importar CSV

  • Drag & drop ou file picker; máx. 5MB
  • PapaParse detecta delimitador (, ou ;) automaticamente
  • Preview da primeira linha após upload
  • Skipável — user pode importar mais tarde

Passo 3 — Mapeamento de Colunas

  • Só aparece se CSV foi importado
  • Mostra primeiras 3 linhas do CSV; user mapeia: Data, Descrição, Valor
  • Opção "Sinal negativo = despesa" (checkbox, default marcado)
  • Preview ao vivo do resultado do mapeamento
  • Configuração guardada para imports futuros (skip automático)

Passo 4 — Categorias

  • Categorias pré-definidas criadas automaticamente:
    • 🛒 Supermercado (keywords: continente, pingo doce, lidl, aldi, mercadona)
    • 🚗 Transportes (keywords: uber, bolt, cp, carris, metro)
    • 📺 Subscrições (keywords: netflix, spotify, apple, amazon, disney)
    • 📱 Telecomunicações (keywords: nos, meo, vodafone, nowo)
    • 💰 Rendimento (keywords: salário, vencimento, ordenado)
  • User pode renomear, mudar cor/ícone, ou criar novas
  • Regras keyword criadas silenciosamente em category_rules

Passo 5 — Despesas Fixas

  • Lista iniciada com "Renda / Hipoteca" como sugestão (vazia, não obrigatória)
  • Cada item: nome, valor, frequência (mensal/semanal/quinzenal/anual), dia
  • Commit para Supabase apenas aqui no "Concluir" — incluindo profiles.balance do Passo 1; até lá tudo vive em localStorage
  • Se o utilizador abandonar antes de "Concluir", é redirecionado para o wizard no próximo acesso (estado recuperado do localStorage se ainda presente)

Dashboard (Home) — Componentes

  1. Saldo card — clicável para editar inline; mostra saldo atual e data de atualização
  2. Banner de alerta (dia crítico)
    • Só aparece se dia crítico ≤ 30 dias
    • Azul warning se > 7 dias; vermelho se ≤ 7 dias
    • Texto: "Dia crítico: 18 Abr — Se não gastares mais nada além das fixas"
  3. Gráfico de linha — projeção 30 dias
    • Eixo X = dias; eixo Y = saldo
    • Linha horizontal a zero
    • Ponto vermelho no dia crítico
    • Área abaixo de zero a vermelho translúcido
  4. Resumo do mês — despesas vs rendimentos, barra de progresso percentual
  5. Últimas transações — 3 rows, link "Ver todas →"

Simulador de Gasto

  • Acessível via tab "Simular" ou floating button no dashboard
  • Input de valor → recálculo instantâneo da projeção (zero writes na DB)
  • Mostra: saldo resultante, novo dia crítico (delta face ao atual)
  • Badge verde "Estás bem nos 30 dias" se gasto não causa problema
  • "Registar gasto" → cria transaction manual (source: 'manual') e decrementa profiles.balance pelo mesmo valor, atualizando balance_updated_at; fecha o simulador e recalcula a projeção com o novo saldo

Estados Vazios

  • Sem CSV: card cinzento "Importa o teu extrato para ver o histórico" + botão
  • Sem despesas fixas: projeção mostra linha flat + sugestão de adicionar fixas
  • Saldo zero / onboarding incompleto: redirect para wizard

6. CSV Import Flow

Fora do onboarding, o flow é um bottom sheet modal (não página separada). Disponível em: Tab Transações → "Importar CSV".

[ 1. Upload ] → [ 2. Mapeamento ]* → [ 3. Preview & Confirmação ] → [ 4. Resultado ]

* Passo 2 só aparece se não existe config guardada, ou se user clica "Alterar mapeamento"

Passo 3 — Preview & Confirmação

  • Mostra contagem total, período (data min → max), e lista scrollável de transações
  • Duplicados detetados por (date, description, amount)
  • Default: ignorar duplicados (checkbox marcado); contagem no botão reflete isso

Passo 4 — Resultado

  • Contagem de transações importadas, agrupadas por categoria
  • "Categorizar agora" → abre lista de transações filtrada por "Sem categoria"

Regras de Categorização Automática

Aplicadas durante o import, por ordem de priority DESC. Case-insensitive, substring match contra description.

7. Lógica de Projeção

Calculada client-side (ou Edge Function). Não guardada em tabela.

Expansão de recurring_items no horizonte de 30 dias

Antes de calcular a projeção, cada recurring_item ativo é expandido para todas as suas ocorrências nos próximos 30 dias. O campo next_date é o ponto de partida; as ocorrências seguintes são derivadas pela frequência:

function occurrences(item, today, horizon=30):
  dates = []
  d = item.next_date
  while d <= today + 30 days:
    if d >= today:
      dates.push(d)
    d = nextOccurrence(d, item.frequency, item.day_of_month, item.day_of_week)
  return dates

nextOccurrence(from, frequency):
  'monthly'   → mesmo dia do mês seguinte
  'weekly'    → from + 7 dias
  'quinzenal' → from + 14 dias  (biweekly = every 14 days)
  'yearly'    → mesmo dia do ano seguinte
Nota: biweekly no enum é renomeado para quinzenal — "biweekly" tem semântica ambígua em inglês; o termo português é inequívoco.

Fórmula da projeção

events = expandir todos os recurring_items ativos nos próximos 30 dias

para cada dia d em [hoje, hoje+30]:
  projeção[d] = profiles.balance
    - SUM(events WHERE type='expense' AND date = d)
    + SUM(events WHERE type='income'  AND date = d)
  // saldo acumula: projeção[d] parte do projeção[d-1], não do balance original
  // isto é: projeção[d] = projeção[d-1] + cashflow[d]

projeção[hoje] = profiles.balance + cashflow[hoje]
projeção[d]    = projeção[d-1]    + cashflow[d]   para d > hoje

dia_critico = MIN(d WHERE projeção[d] < 0)
            = undefined se projeção nunca negativa nos 30 dias
Nota: A projeção usa apenas recurring_items — transações históricas do CSV não entram na projeção (são passado, não futuro). Transações futuras manuais são v2.

Convenção de sinais

Existem duas convenções no schema — esta divergência é intencional e documentada:

TabelaConvençãoRazão
transactions.amountNegativo = despesa; positivo = rendimentoEspelha o extrato bancário diretamente; facilita import CSV
recurring_items.amountSempre positivo; type define direçãoEvita erros de sinal em formulários; valor introduzido pelo user é sempre positivo

Qualquer utilitário de formatação ou soma deve verificar de qual tabela vem o dado antes de aplicar sinal.

Simulador

projeção_simulada[d] = projeção[d] - gasto_simulado (para todo d >= hoje)
dia_critico_simulado = MIN(d WHERE projeção_simulada[d] < 0)

8. Questões em Aberto

#QuestãoImpactoStatus
1 Múltiplas contas bancárias em v1 ou apenas uma? Schema (tabela accounts separada vs balance direto em profiles) Decidido: uma conta (MVP)
2 Notificações push quando se aproxima o dia crítico? Requer Service Worker + push API Adiado para v2

5 "Resumo do mês" — fonte de dados: transactions históricas, recurring_items, ou ambas? Display e estado vazio para users sem CSV Decidido: apenas transactions do mês corrente; se vazio, card mostra "Sem transações este mês — importa um extrato"
6 Config tab: ecrã para editar recurring items e categorias pós-onboarding? Funcionalidade implícita mas não desenhada A desenhar antes de implementar Config tab
